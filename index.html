PadSpectra

Transformer-based spectral padding for leaf and soil spectra (350–2500 nm)

This page explains how to use the web app, the input rules, and the API contract.

What PadSpectra Does

PadSpectra reconstructs missing wavelength regions from incomplete spectra by combining spectral values with auxiliary metadata (sample type, spectral type, spectrometer, and geometric configuration). It is designed for heterogeneous datasets and low/partial-range spectrometers.

Web App Workflow

  1. Upload a Spectra CSV (required).
  2. Upload a Property CSV (optional if property override is fully selected).
  3. Select spectra indices to visualize.
  4. Set keep wavelength ranges (optional mask).
  5. Run PadSpectra Padding for all rows.
  6. Download padded spectra CSV.

Data Upload Rules

Spectra CSV

  • Header must contain numeric wavelength columns.
  • Each row is one spectrum.
  • Values should be numeric; empty/NaN values are allowed.
  • Extra non-wavelength columns may be ignored for model input.
  • Model input expects 2150 values (43 × 50); if 2151, last value is dropped for inference.
  • Masked regions are represented by value 0 for model input.

Property CSV

  • Each row corresponds to a spectrum (row-order alignment by default).
  • Required columns: sample, spectral type, spectrometer, geometric configuration.
  • Values should be strings.
  • Property CSV is optional when user-specified property override is fully active.

Keep Wavelength Ranges Rules

  • Accepts 1 or 2 ranges, e.g. 350-2000 or 400-900,1100-2300.
  • Endpoints must be within 350–2500.
  • All endpoints must be divisible by 50.
  • Total kept span must be at least 500 nm.
  • Three or more ranges are not allowed.
  • Ranges must not overlap.
  • If two ranges are used, the gap between them must be ≤ 400 nm.

API

Endpoint: POST /api/padspectra/predict_all

Purpose: run padding for all spectra rows in one call.

Request Schema

{
  "wavelengths": [ ... 2150 numbers ... ],
  "spectra_model": [
    [ ... 2150 numbers ... ],
    [ ... 2150 numbers ... ]
  ],
  "properties": [
    {
      "sample": "leaf",
      "spectral type": "reflectance",
      "spectrometer": "asd",
      "geometric configuration": "directional - directional"
    }
  ],
  "config": {
    "bandnum": 43,
    "bandwidth": 50,
    "maxsentences": 4,
    "maxwords": 6,
    "pad_index": 35
  }
}

Response Schema

{
  "padded": [
    [ ... 2150 numbers ... ],
    [ ... 2150 numbers ... ]
  ]
}

curl Example

curl -X POST http://localhost:3000/api/padspectra/predict_all \
  -H "Content-Type: application/json" \
  -d @payload.json

Deployment Note

If using Docker Compose, run: docker compose up -d --build

Web is exposed on port 3000. Inference service stays internal and is called by the web app.
